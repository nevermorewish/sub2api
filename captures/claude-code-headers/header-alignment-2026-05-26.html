4. 网关逻辑层差异

关注点本地上游
OAuth 客户端 beta 补齐逻辑（getBetaHeader） 直接透传 clientBetaHeader；缺省时回落到 DefaultBetaHeader / HaikuBetaHeader。真实 CLI 不带 oauth-2025-04-20，不需要补齐。 如果 clientBetaHeader 不含 oauth-2025-04-20，会在 claude-code-20250219 之后强制插入 oauth-2025-04-20。基于旧抓包的假设。
Mimicry 路径 requiredBetas 起点 非 haiku → FullClaudeCodeMimicryBetas()；haiku → {BetaInterleavedThinking} 非 haiku → FullClaudeCodeMimicryBetas()；haiku → {BetaOAuth, BetaInterleavedThinking}
cache_control TTL 顺序冲突规整 新增 normalizeAnthropicCacheControlTTLOrder()：如果请求中先出现 ttl=5m、后出现 ttl=1h，统一强制升级到 1h，规避 Anthropic 的 cache TTL 顺序冲突报错。 无此逻辑。
billing block 格式 cc_version=2.1.132.{fp}; cc_entrypoint=cli; cch=00000; cc_version=2.1.92.{fp}; cc_entrypoint=cli; cch=00000;

5. 为什么这些差异会影响计费判定

  • User-Agent / X-Stainless-* 不匹配真实 CLI：Anthropic 反作弊会把 (claude-cli/X.Y.Z, X-Stainless-OS, X-Stainless-Arch, X-Stainless-Runtime-Version, X-Stainless-Package-Version) 作为指纹组合判定。任一组合越偏离真实 CLI 当前流量，越容易被识别为第三方调用 → Third-party apps now draw from your extra usage。
  • anthropic-beta 集合不匹配：缺失 claude-code-20250219 或带上真实 CLI 已经不再发送的 oauth-2025-04-20 都会被识别为非 CC 流量。本地代码已经按真实抓包的精确顺序输出。
  • cc_entrypoint 必须和 User-Agent (external, X) 内的 X 保持一致：真实 CLI 这两处都是 cli。SDK 派生的进程会变成 sdk-cli，但本仓伪装的是交互 CLI，所以全用 cli。
  • extended-cache-ttl-2025-04-11 是本地策略性扩展：真实 CLI 不带它，但本仓有"开启后强制注入 1h 缓存断点"的开关；当该开关开启时如果不带这个 beta，Anthropic 会拒绝 ttl=1h。所以把它附加在 FullClaudeCodeMimicryBetas() 列表末尾。
