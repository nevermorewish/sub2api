Claude Code TLS Fingerprint 生成逻辑

结论：Claude Code 源码没有手写 TLS 指纹表。真实 TLS ClientHello 由运行时的网络栈生成： Claude Code -> @anthropic-ai/sdk -> globalThis.fetch -> Node.js/undici -> OpenSSL。 因此不能只按“Node 版本参数”推导，必须以真实 ClientHello 捕获值为准。

两个“指纹”不要混淆

名称 来源 用途 是否 TLS 指纹
cc_version 后缀 3 位 fingerprint D:\Claude\claude-code-source-code\src\utils\fingerprint.ts Claude Code 归因头：SHA256(salt + msg[4] + msg[7] + msg[20] + version)[:3] 不是
JA3 / JA4 / TLS ClientHello Node.js/undici/OpenSSL 在 HTTPS 握手时产生 网关模拟上游 HTTPS TLS ClientHello 是

如果“按照参数生成指纹”不对，常见原因是把 Claude Code 的归因 fingerprint 当成 TLS fingerprint， 或者只配置了 cipher/curve 等字段但没有配置真实的 extensions 顺序。

从 ClientHello 到 JA3 的计算逻辑

JA3 Raw 组装

function isGrease(v) {
  return (v & 0x0f0f) === 0x0a0a && (v >> 8) === (v & 0xff);
}

function ja3Raw(fp) {
  const tlsVersion = 771; // JA3 uses legacy ClientHello version for this capture.
  const ciphers = fp.cipherSuites.filter(v => !isGrease(v) && v !== 0x00ff);
  const extensions = fp.extensions.filter(v => !isGrease(v));
  const curves = fp.curves.filter(v => !isGrease(v));
  const pointFormats = fp.pointFormats;
  return [
    tlsVersion,
    ciphers.join("-"),
    extensions.join("-"),
    curves.join("-"),
    pointFormats.join("-"),
  ].join(",");
}

JA3 Raw 里第一个版本是 771，即 legacy ClientHello version 0x0303。 supported_versions 扩展仍然声明实际支持 [772, 771]，即 TLS 1.3 + TLS 1.2。 不要把 JA3 第一段改成 772。

为什么参数生成会偏差

问题 表现 修正
extensions 未配置 后端使用默认 [0,65037,23,65281,10,11,35,16,5,13,18,51,45,43]，和捕获值不同 显式写入 [0,23,65281,10,11,35,16,5,13,18,51,45,43,21]
把 ECH 65037 加进当前 Windows 捕获 JA3 扩展段多出 65037，hash 变化 本次捕获没有 ECH，不能凭默认模板添加
漏掉 padding 21 JA3 扩展末尾缺失 21 保留扩展末尾的 padding
只按 Node 大版本推导 同为 Node 24，不同 OS/构建/代理路径仍可能不同 以真实 Claude Code 请求采集为准
ALPN 误配为 ["h2","http/1.1"] JA4/HTTP2 指纹变化 本次捕获为 ["http/1.1"]

实现规则

  1. 先用真实 Claude Code 进程向采集服务发请求，捕获原始 ClientHello。
  2. 解析 cipher suites、extensions、supported groups、point formats、signature algorithms、ALPN、supported versions、key share、PSK modes。
  3. JA3 计算时移除 GREASE 和 SCSV，但 Sub2API profile 里保留运行时真实可发送的字段顺序。
  4. 把 extensions 写成 profile 的正式字段，不能只放到注释。
  5. Sub2API 用 uTLS 按 profile 顺序构造 ClientHello；任何未配置字段都会回退默认值。
  6. Claude Code、Node.js、OpenSSL 或代理路径变更后重新采集。

可直接用于采集后的转换函数

function profileFromCaptured(fp, key = "windows_x64_node_v2430") {
  return {
    [key]: {
      name: fp.name || "Claude Code captured TLS profile",
      enable_grease: Boolean(fp.enable_grease),
      cipher_suites: fp.cipher_suites,
      curves: fp.curves,
      point_formats: fp.point_formats,
      signature_algorithms: fp.signature_algorithms,
      alpn_protocols: fp.alpn_protocols,
      supported_versions: fp.supported_versions,
      key_share_groups: fp.key_share_groups,
      psk_modes: fp.psk_modes,
      extensions: fp.extensions
    }
  };
}
